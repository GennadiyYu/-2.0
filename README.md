# AI-ассистент руководителя digital-агентства

Готовый Node.js-проект для Telegram-бота, который:

- отвечает как AI-ассистент руководителя digital-агентства;
- анализирует загруженные файлы;
- позволяет задавать вопросы по документам;
- хранит историю диалога и список документов по каждому чату.

## Что умеет

- рекомендации по управлению агентством, продажам, клиентскому сервису и команде;
- разбор PDF, DOCX, XLSX, XLS, CSV, TXT, MD, JSON;
- summary, риски, рекомендации, вопросы на уточнение;
- ответ по содержимому уже загруженных документов;
- команды `/start`, `/help`, `/files`, `/reset`.

## Стек

- Node.js 20+
- Express
- Telegram Bot API через webhook
- OpenAI Node SDK
- Локальное JSON-хранилище для MVP

## Быстрый старт

### 1. Установка

```bash
git clone <your-repo-url>
cd agency-exec-ai-bot
npm install
```

### 2. Переменные окружения

Создайте `.env` на основе примера:

```bash
cp .env.example .env
```

Заполните значения:

```env
PORT=3000
BASE_URL=https://your-domain.com
TELEGRAM_BOT_TOKEN=your_telegram_bot_token
OPENAI_API_KEY=your_openai_api_key
OPENAI_MODEL=gpt-4.1-mini
ADMIN_CHAT_IDS=
MAX_CHUNKS_PER_FILE=30
MAX_CHUNK_LENGTH=1800
TOP_K_CHUNKS=8
```

### 3. Локальный запуск

```bash
npm run dev
```

Проверка:

```bash
http://localhost:3000/health
```

## Настройка Telegram webhook

Бот сам вызывает `setWebhook()` при старте, если указан `BASE_URL`.

Webhook-адрес:

```bash
https://your-domain.com/telegram/webhook
```

## Как пользоваться

1. Запустите бота.
2. Напишите `/start`.
3. Загрузите документ.
4. Задавайте вопросы:
   - `Сделай summary`
   - `Какие риски ты видишь?`
   - `Что здесь слабое место?`
   - `Сформулируй рекомендации для руководителя`
   - `Подготовь вопросы к аккаунт-команде`

## Как работает анализ файлов

### Поддерживаемые форматы

- PDF
- DOCX
- XLSX / XLS
- CSV
- TXT
- MD
- JSON
- HTML / XML

### Логика

1. Пользователь отправляет документ в Telegram.
2. Бот скачивает файл через Telegram API.
3. Из файла извлекается текст.
4. Текст режется на фрагменты.
5. Фрагменты сохраняются в хранилище.
6. Когда пользователь задаёт вопрос, бот подбирает релевантные куски текста и передаёт их в модель.

## Команды

- `/start` — приветствие и описание возможностей
- `/help` — краткая инструкция
- `/files` — список файлов в текущем чате
- `/reset` — очистка истории и загруженных файлов

## Важно

Сейчас в проекте используется **локальное JSON-хранилище**. Это хороший вариант для MVP и тестов, но для продакшена лучше заменить его на:

- PostgreSQL / Supabase — для истории и пользователей
- S3 / Cloudflare R2 / Supabase Storage — для файлов
- Redis — для сессий и очередей
